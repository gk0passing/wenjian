下面按你的定义整理：OCR 小模型 = 不以 VLM/LLM 为核心的专用 OCR 模型，包括传统检测-识别两阶段、CTC/Attention/Transformer 解码器、公式/表格/版面专用模型、轻量化部署框架。像 PaddleOCR-VL、HunyuanOCR、Surya2、DeepSeek-OCR 这类 OCR-VLM/LLM 解码路线不放进主线，只作为边界对照。

1. 总体判断

2024–2026 年 OCR 小模型没有被 VLM 替代，反而形成了更清晰的工业路线：小模型负责高精度文字定位、字符级识别、结构化解析和端侧部署；VLM 更适合理解、问答和复杂语义推理。小模型的核心进展主要集中在四点：

检测端：从复杂模块堆叠转向更简单、更鲁棒的分割/距离图/自适应卷积方法，同时利用合成数据、无标注真实数据做预训练。
识别端：CTC 路线重新变强，SVTRv2 证明在设计合理时，CTC 可以在速度和精度上超过很多 Encoder-Decoder 识别器。
工业端：PaddleOCR 形成了 PP-OCRv5/v6、PP-StructureV3、PP-FormulaNet 等完整产品线，覆盖通用 OCR、文档解析、表格、公式和部署。
部署端：RapidOCR、OpenOCR、ONNX Runtime、OpenVINO、TensorRT 等生态让小模型继续适合 CPU、移动端、边缘设备和私有化场景。

3. 工业界重点：PaddleOCR 系列是主线
3.1 PP-OCRv5：5M 参数级通用 OCR 小模型

PP-OCRv5 的重点不是单纯换网络，而是强调数据质量、数据难度、数据多样性对传统 OCR 两阶段模型上限的影响。PaddleOCR 3.0 报告中，PP-OCRv5 被定位为高精度轻量 OCR 模型，支持简体中文、繁体中文、英文、日文、拼音等多语种，并提供 server/mobile 版本。2026 年的 PP-OCRv5 论文进一步把它定义为 5M 参数级模型，强调在足够高质量数据下，小模型仍可逼近甚至对抗大规模 VLM 的 OCR 表现。

启发：如果你做 OCR 数据校验或小模型训练，不要只讲模型结构，要重点讲数据闭环：错标过滤、难例挖掘、多 teacher 一致性、低质量样本剔除、字符级置信度和版面规则，这正是 PP-OCRv5 这种路线背后的核心。

3.2 PP-OCRv6：2026 年工业 OCR 小模型的代表

PP-OCRv6 是目前工业界最值得重点看的 OCR 小模型路线之一。它不是 VLM，而是继续沿用 OCR 专用 pipeline，但把 backbone、检测 neck、识别 neck 统一升级。官方资料显示，PP-OCRv6 设计了 PPLCNetV4 作为统一轻量 backbone，引入 MetaFormer 风格结构和结构重参数化；检测侧使用 RepLKFPN；识别侧使用 EncoderWithLightSVTR；并提供 tiny、small、medium 三个尺寸，参数量从 1.5M 到 34.5M，覆盖边缘设备、移动端、桌面端和服务器部署。

PP-OCRv6 的一个重要信号是：工业界并没有放弃传统 OCR 小模型，而是在用更强的数据、更轻量的 backbone、更合理的检测/识别模块继续压榨上限。官方文档称，PP-OCRv6_medium 相比 PP-OCRv5_server 识别精度提升 5.1%，检测 Hmean 提升 4.6%，GPU 推理速度提升 2.37 倍，并且 medium/small 支持 50 种语言，tiny 支持 49 种语言。

可以重点研究的点：

PPLCNetV4：轻量 CNN + MetaFormer 风格 token mixing。
RepLKFPN：大核卷积和结构重参数化，增强检测多尺度特征。
LightSVTR：将 SVTR 思想压缩到工业识别器中。
tiny/small/medium 分层：对应端侧、移动端、服务器不同延迟/精度要求。
多语言统一模型：比每种语言单独训练更适合工业部署。
4. 学术界重点：SVTRv2 是识别端最关键工作

SVTRv2 的核心观点是：CTC 并没有过时，只是以前的视觉特征和上下文建模不够好。它提出 MSR、FRM、SGM 等模块，让 CTC 模型在保持快速推理的同时，补上不规则文本、长文本和语言上下文建模的短板。尤其是 SGM 属于训练增强模块，推理时可以移除，因此不会明显增加部署成本。

这对 OCR 小模型非常重要，因为工业部署最怕自回归解码慢、显存高、延迟不稳定。SVTRv2 说明：只要 backbone、特征重排和训练监督设计得好，CTC 路线仍然可以同时做到高精度和高速度。

可以把识别端路线分成三类：

路线	代表	特点	适合场景
CTC 高速识别	SVTRv2、PP-OCRv6 recognition	非自回归、速度快、部署友好	工业 OCR、移动端、CPU/ONNX 推理
Encoder-Decoder / Attention	SceneXL、One2Seq	上下文建模强，但推理通常更复杂	不规则文本、长文本、高精度场景
扩散/迭代解码	MDiff4STR	用 mask diffusion 做字符序列生成，少步去噪降低延迟	研究型方向，可探索鲁棒识别
5. 文档结构化方向：从 OCR 到 Document Parsing

2024–2026 年另一个明显趋势是：小模型不只做“框+字”，而是开始做文档结构化，包括版面分析、表格识别、公式识别、图表理解和阅读顺序恢复。

PP-StructureV3 是工业界代表，它可以把文档图片或 PDF 转成 Markdown，并覆盖 layout detection、table recognition、formula recognition、chart understanding、multi-column reading order recovery 等模块。它仍然是更偏 OCR/document parsing 的专用系统，而不是通用 VLM 问答模型。

公式方向，PP-FormulaNet 和 UniMERNet 是重点。PP-FormulaNet 分为高精度版和高效率版，官方论文称 PP-FormulaNet-L 比 UniMERNet 提升 6% 以上，PP-FormulaNet-S 推理速度快 16 倍以上；UniMERNet 则通过 UniMER-1M 数据集推动真实场景公式识别。

表格方向，工业上可以看 SLANet/SLANet_plus，学术上可以看 TableSeq。SLANet_plus 是 PaddleX/PaddleOCR 体系里的轻量表格结构识别模型，模型约 6.9M 参数；TableSeq 则尝试把结构、内容、布局统一成一个序列，不依赖外部 OCR 或复杂后处理。

6. 开源与部署生态
RapidOCR

RapidOCR 的价值不在于提出新结构，而在于把 PaddleOCR 模型转换到 ONNX，并支持 Python、C++、Java、C# 等多语言调用，适合离线部署、跨平台部署和私有化交付。它还支持 ONNX Runtime、OpenVINO、Paddle、PyTorch、MNN、TensorRT 等推理后端。

适合场景：CPU OCR、桌面软件、移动端、企业私有化、没有 GPU 的业务系统。

OpenOCR

OpenOCR 更偏学术和工业中间层，由复旦 FVL OCR 团队维护，覆盖文本检测、文本识别、公式识别、表格识别、文档解析等任务，并提供统一训练、评测和复现框架。它也集成了 OpenDoc-0.1B、UniRec-0.1B 等文档解析/统一识别模型。

适合场景：做研究复现、改模型结构、跑 SVTRv2/MDiff4STR/公式表格模型、做秋招项目 baseline。

7. 技术趋势总结
趋势一：小模型不再只拼 backbone，而是“结构 + 数据 + 训练策略”一起优化

PP-OCRv5/v6 都说明，OCR 小模型的提升不只是换网络，而是同时依赖数据清洗、难例构造、多语言覆盖、工业场景增强、结构重参数化和部署优化。

趋势二：CTC 路线重新成为强基线

SVTRv2 最大的意义是把 CTC 从“速度快但精度弱”的印象里拉回来。它说明，只要视觉特征足够强，训练时加入语义指导，CTC 也可以在复杂场景文字识别上接近甚至超过 Encoder-Decoder 模型。

趋势三：检测更重视鲁棒性和数据分布

KAC、FreeReal、ODM、TextDS 都在解决一个问题：真实场景文字的形态、背景、模糊、弯曲、遮挡、风格变化太大，检测模型需要更好的数据适配和分布泛化，而不只是更大的网络。

趋势四：OCR 小模型向“文档结构化组件”演进

表格、公式、版面、阅读顺序恢复已经成为 OCR 小模型的重要组成部分。未来工业 OCR 不会只是输出文字，而是输出 Markdown、HTML、LaTeX、JSON、表格结构和版面层级。

8. 选型建议

如果你要做通用工业 OCR 小模型项目，优先看：

PP-OCRv6：最适合作为工业主线，包含检测、识别、多语言、部署分层。
SVTRv2 / OpenOCR：适合作为识别端改进 baseline，尤其适合做论文型或秋招项目。
RapidOCR：适合做端侧、CPU、私有化部署。
PP-StructureV3 + PP-FormulaNet + SLANet_plus：适合做文档解析、表格、公式、PDF 转 Markdown。
KAC / FreeReal / ODM / TextDS：适合做检测鲁棒性、弱监督预训练、工业场景泛化研究。

如果你要做一个有深度的 OCR 小模型项目，可以围绕这个方向设计：

“面向工业回流数据的轻量 OCR 小模型训练与数据闭环系统”：以 PP-OCRv6/SVTRv2 为 backbone，结合多 teacher 数据校验、字符级置信度、检测框一致性、难例挖掘、伪标签过滤和端侧量化部署，目标是在不使用 VLM 的情况下提升小模型在复杂业务场景下的识别精度、鲁棒性和推理速度。
