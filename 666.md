# MDiff4STR 方法细节总结

## 1. 方法总览

MDiff4STR 的核心思想是：将场景文本识别任务从传统的“逐字符生成”转化为“图像条件下的多步文本去噪与纠错”。

传统自回归场景文本识别模型通常按照从左到右的顺序逐个预测字符，而 MDiff4STR 使用 **Mask Diffusion Model, MDM**，从一串被 mask 或被污染的字符 token 出发，在图像特征的条件下逐步恢复出完整文本序列。

其整体流程可以概括为：

```text
输入文本图像
    ↓
视觉编码器提取图像特征
    ↓
构造被 mask 或被替换污染的文本序列
    ↓
Mask Diffusion Decoder 进行去噪或纠错
    ↓
输出最终识别文本
```

MDiff4STR 的关键创新主要包括：

1. 使用 Mask Diffusion Model 进行场景文本识别；
2. 设计多种训练 mask 策略，缓解训练与推理噪声分布不一致；
3. 提出 Token-Replacement Noise，增强模型纠正错误 token 的能力；
4. 使用多步 low-confidence remask 或 block low-confidence remask 完成高效推理。

---

## 2. 整体架构

MDiff4STR 由两个主要模块组成：

* **视觉编码器 Visual Encoder**
* **掩码扩散解码器 Mask Diffusion Decoder**

### 2.1 输入与目标

输入是一张文本图像：

```math
X \in \mathbb{R}^{H \times W \times 3}
```

对应的真实文本标签为：

```math
Y = \{y_1, y_2, \dots, y_L\}
```

其中：

* (L) 表示文本序列长度；
* (y_i) 表示第 (i) 个字符；
* (V) 表示字符词表。

模型目标是根据图像 (X)，预测出完整文本序列 (Y)。

---

### 2.2 视觉编码器

论文采用 **SVTRv2** 作为视觉编码器，用于从文本图像中提取视觉特征：

```math
F_v = \text{SVTRv2}(X)
```

其中：

```math
F_v \in \mathbb{R}^{H/8 \times W/4 \times D}
```

(F_v) 是图像的视觉特征表示，后续会作为条件信息输入到文本去噪解码器中。

---

### 2.3 文本 token 嵌入

对于被污染后的文本序列 (Y_m)，模型首先通过字符嵌入层将其转换为 token 表示：

```math
T_m = \text{Embedding}(Y_m)
```

其中：

```math
T_m \in \mathbb{R}^{L \times D}
```

这里的 (Y_m) 可以是被 mask 的文本序列，也可以是经过 token replacement 后的 corrupted sequence。

---

### 2.4 Mask Diffusion Decoder

Mask Diffusion Decoder 接收两个输入：

1. 图像特征 (F_v)
2. 文本 token 表示 (T_m)

然后输出文本 token 的隐表示：

```math
\tilde{T} = \text{MDiffDecoder}(F_v, T_m)
```

最后通过分类器得到每个位置的字符预测：

```math
\tilde{Y} = \text{Classifier}(\tilde{T})
```

Decoder 内部主要包括：

* token self-attention；
* cross-attention to visual features；
* feed-forward network / MLP。

其中 cross-attention 使文本 token 在恢复时能够关注图像中的对应视觉区域。

---

## 3. Vanilla MDM：基础 mask 去噪

最基础的 Mask Diffusion Model 训练方式类似 BERT 的 masked token prediction。

给定真实文本序列 (Y)，先使用训练 mask 策略 (\mathcal{M}_T) 生成被 mask 的文本序列：

```math
Y_m = \mathcal{M}_T(Y)
```

然后模型根据图像特征和被 mask 的文本序列恢复原始文本：

```math
F_v = \text{SVTRv2}(X)
```

```math
T_m = \text{Embedding}(Y_m)
```

```math
\tilde{T} = \text{MDiffDecoder}(F_v, T_m)
```

```math
\tilde{Y} = \text{Classifier}(\tilde{T})
```

直观理解是：

> 模型看到一部分字符和一部分 `[MASK]`，再结合图像内容，预测被遮盖的字符。

不过，vanilla MDM 直接用于场景文本识别时存在两个问题：

1. 训练阶段通常使用随机 mask，而推理阶段使用的是全 mask 起步和结构化 remask，导致训练与推理噪声分布不一致；
2. 模型可能对错误字符产生过高置信度，导致后续 remask 无法发现并修正错误。

MDiff4STR 的主要方法设计就是围绕这两个问题展开的。

---

## 4. 推理过程：多步 denoising

MDiff4STR 在推理时从全 `[MASK]` 序列开始：

```math
Y_m^1 = [\text{MASK}]^{\otimes L}
```

第 (i) 步推理时，模型先将当前 token 序列嵌入：

```math
T_m^i = \text{Embedding}(Y_m^i)
```

然后进行一次文本预测：

```math
\tilde{T}^i = \text{MDiffDecoder}(F_v, T_m^i)
```

```math
\tilde{Y}^i = \text{Classifier}(\tilde{T}^i)
```

接着根据推理 remask 策略 (\mathcal{M}_I)，把部分 token 重新 mask，得到下一步输入：

```math
Y_m^{i+1} = \mathcal{M}_I(\tilde{Y}^i)
```

经过 (K) 步去噪后，得到最终预测结果：

```math
\tilde{Y}^K
```

论文最终采用：

```text
K = 3
Decoder 层数 N = 6
```

作为速度和精度之间的折中配置。

---

## 5. 多种 remask 策略

MDiff4STR 的一个重要特点是，不同的 remask 策略可以对应不同的文本解码范式。

---

### 5.1 Full Mask：并行解码

Full Mask 表示推理开始时所有位置都是 `[MASK]`：

```text
[MASK] [MASK] [MASK] [MASK] ...
```

模型一次性预测整串文本。

这种方式类似并行解码，速度最快，但缺少后续修正能力。

论文中对应：

```text
MDiff-PD
```

即 parallel decoding 形式。

---

### 5.2 Right Remask / Left Remask：自回归式去噪

Right Remask 可以模拟从左到右的自回归解码。

例如：

```text
Step 1: [MASK] [MASK] [MASK] [MASK]
Step 2: A      [MASK] [MASK] [MASK]
Step 3: A      B      [MASK] [MASK]
Step 4: A      B      C      [MASK]
```

这种方式逐渐固定左侧 token，继续预测右侧 token。

Left Remask 则是反方向，从右到左逐步确定字符。

论文中对应：

```text
MDiff-AR
```

即 auto-regressive decoding 形式。

需要注意的是，MDiff-AR 和传统 ARM 不完全相同。传统 ARM 每次只预测下一个字符，而 MDiff-AR 仍然是在扩散去噪框架下预测整串文本，只是通过 remask 策略表现出类似自回归的顺序。

---

### 5.3 Cloze Remask：BERT-like refinement

Cloze Remask 类似完形填空：

```text
A [MASK] C [MASK] E
```

模型根据左右上下文和图像特征恢复被 mask 的字符。

这种方式对应 BERT-like refinement，论文中称为：

```text
MDiff-Re
```

它的优点是可以利用双向上下文，缺点是如果已有 token 本身是错误的，模型可能会被错误上下文误导。

---

### 5.4 Low-Confidence Remask

Low-Confidence Remask 是 MDiff4STR 中比较核心的推理策略。

每一步预测后，模型会为每个 token 计算置信度，然后将低置信度 token 重新 mask，再送入下一轮去噪。

示意如下：

```text
Step 1 prediction:
A  R  T  E
0.98 0.95 0.42 0.88

低置信度 token:
T

Remask 后:
A  R  [MASK]  E
```

然后模型在下一步重新预测被 mask 的位置。

这种方式的直觉是：

> 模型先猜一遍整串文本，再把自己不确定的位置擦掉重写。

论文中对应：

```text
MDiff-LC
```

LC 的优点是灵活，不受从左到右顺序限制，任何位置都可以被重新修正。

---

### 5.5 Block Low-Confidence Remask

普通 Low-Confidence Remask 可能出现一个问题：某些 token 因为长期低置信度，会在多轮推理中反复被 mask，导致生成过程陷入停滞。论文称这种现象为：

```text
confidence trap
```

为解决这个问题，MDiff4STR 提出 Block Low-Confidence Remask。

其做法是将文本序列划分为若干 block，只在局部 block 内选择低置信度 token 进行 remask。

block size 设置为：

```math
L / K
```

其中：

* (L) 是文本长度；
* (K) 是去噪步数。

论文中对应：

```text
MDiff-BLC
```

BLC 的作用是：

1. 避免某些 token 被反复 mask；
2. 让不同局部区域都有机会被修正；
3. 提高多步推理的稳定性。

论文最终更推荐 BLC，因为它在准确率和推理稳定性之间表现较好。

---

## 6. Training Mask Strategies：缓解 noising gap

vanilla MDM 通常只使用 random mask 训练。

但 MDiff4STR 的推理过程并不是随机 mask，而是从全 mask 开始，并经过多种结构化 remask 策略。也就是说：

```text
训练阶段噪声：random mask
推理阶段噪声：full mask + structured remask
```

二者不一致，导致训练和推理之间存在噪声分布差异。论文称之为：

```text
training-inference noising gap
```

为缓解这个问题，MDiff4STR 在训练时引入多种 mask/remask 策略，使训练阶段的噪声分布更加接近推理阶段。

训练中使用的 mask 策略包括：

1. Random Mask
2. Full Mask
3. Right Remask
4. Left Remask
5. Cloze Remask
6. Low-Confidence Remask
7. Block Remask

训练时，每个样本会从这些策略中均匀采样一种，用来构造训练输入。

这样做的好处是：

* 模型训练时就见过推理阶段可能出现的噪声模式；
* 模型对 full mask 起步更加适应；
* 模型对结构化 remask 的鲁棒性更强；
* 多种推理范式都能从统一训练过程中受益。

可以概括为：

> MDiff4STR 通过让训练噪声覆盖推理噪声，缓解了 training-inference noising gap。

---

## 7. Token-Replacement Noise：纠正高置信错误

仅仅使用 mask 去噪还不够，因为推理时模型产生的错误并不一定表现为 `[MASK]`。

更常见的问题是：

```text
某个字符已经被预测出来，而且模型对它非常自信，但它其实是错的。
```

例如：

```text
预测字符: F
置信度: 0.95
真实字符: E
```

如果使用 low-confidence remask，由于该错误字符置信度很高，它不会被重新 mask，因此后续步骤也很难修正它。

这就是 MDiff4STR 要解决的第二个核心问题：

```text
over-confidence on incorrect tokens
```

---

### 7.1 TRN 的基本思想

MDiff4STR 提出 **Token-Replacement Noise, TRN**。

TRN 不只是把字符替换成 `[MASK]`，而是随机把部分真实字符替换成其他错误字符。

例如真实文本为：

```text
APPLE
```

经过 token replacement 后可能变为：

```text
APFLE
```

这里 `F` 是一个错误 token，但它不是 `[MASK]`，看起来像一个正常字符。

模型需要根据图像特征和上下文，把它纠正回正确字符：

```text
APFLE → APPLE
```

---

### 7.2 TRN 的公式表示

给定真实文本序列 (Y)，随机选择 (l_2) 个位置，并将这些位置的字符替换为词表中的其他字符，得到 corrupted sequence：

```math
Y_r \in V^L
```

然后通过 embedding 得到：

```math
T_r = \text{Embedding}(Y_r)
```

输入 decoder：

```math
\tilde{T} = \text{MDiffDecoder}(F_v, T_r)
```

```math
\tilde{Y} = \text{Classifier}(\tilde{T})
```

TRN 的关键作用是让模型学会：

1. 不盲目信任已有 token；
2. 根据图像特征检查 token 是否正确；
3. 根据上下文修正错误字符；
4. 处理高置信但错误的预测结果。

---

## 8. 损失函数设计

MDiff4STR 使用两个训练目标：

1. denoising loss
2. correction loss

最终总损失为：

```math
\mathcal{L}_{total}
=
\mathcal{L}_{denoising}
+
\mathcal{L}_{correction}
```

---

### 8.1 Denoising Loss

Denoising loss 用于 mask denoising 训练。

它只监督被 mask 的位置：

```math
\mathcal{L}_{denoising}
=
-\frac{1}{l_1}
\sum_{i=1}^{L}
\mathbf{1}[Y_{l_1}^{i}=M]
\log p_{\theta}(Y^i \mid Y_{l_1})
```

其中：

* (Y_{l_1}) 表示经过 mask 后的序列；
* (l_1) 表示被 mask 的 token 数量；
* (M) 表示 `[MASK]` token；
* (\mathbf{1}[\cdot]) 是指示函数；
* 只有被 mask 的位置才计算 loss。

这个 loss 主要训练模型：

```text
根据图像和上下文补全缺失字符。
```

---

### 8.2 Correction Loss

Correction loss 用于 Token-Replacement Noise 训练。

它对整个序列都进行监督：

```math
\mathcal{L}_{correction}
=
-\frac{1}{L}
\sum_{i=1}^{L}
\log p_{\theta}(Y^i \mid Y_{l_2})
```

其中：

* (Y_{l_2}) 表示经过 token replacement 后的 corrupted sequence；
* (l_2) 表示被替换的 token 数量；
* (L) 表示文本长度。

这里不是只监督被替换的位置，而是监督整个序列。

原因是：

```text
推理阶段模型并不知道哪些 token 是错的。
```

因此，训练时必须让模型对所有 token 都具备检查和纠错能力。

---

## 9. 训练流程总结

MDiff4STR 的训练可以理解为两条分支联合优化。

---

### 9.1 Mask Denoising 分支

该分支用于学习从 `[MASK]` 中恢复字符。

流程如下：

```text
真实文本 Y
    ↓
训练 mask 策略 M_T
    ↓
masked sequence Y_m
    ↓
Embedding
    ↓
Mask Diffusion Decoder
    ↓
预测文本 Y_hat
    ↓
Denoising Loss
```

对应公式为：

```math
Y_m = \mathcal{M}_T(Y)
```

```math
T_m = \text{Embedding}(Y_m)
```

```math
\tilde{T} = \text{MDiffDecoder}(F_v, T_m)
```

```math
\tilde{Y} = \text{Classifier}(\tilde{T})
```

---

### 9.2 Correction 分支

该分支用于学习纠正错误 token。

流程如下：

```text
真实文本 Y
    ↓
Token-Replacement Noise
    ↓
corrupted sequence Y_r
    ↓
Embedding
    ↓
Mask Diffusion Decoder
    ↓
预测文本 Y_hat
    ↓
Correction Loss
```

对应公式为：

```math
Y_r = \text{TokenReplace}(Y)
```

```math
T_r = \text{Embedding}(Y_r)
```

```math
\tilde{T} = \text{MDiffDecoder}(F_v, T_r)
```

```math
\tilde{Y} = \text{Classifier}(\tilde{T})
```

---

## 10. 推理流程总结

推理阶段从全 `[MASK]` 开始，通过多轮预测和 remask 得到最终文本。

流程如下：

```text
Step 1:
[MASK] [MASK] [MASK] [MASK]
        ↓
模型预测整串文本

Step 2:
根据置信度或 block 策略重新 mask 部分 token
        ↓
模型再次预测整串文本

Step 3:
继续 remask 和预测
        ↓
输出最终识别结果
```

形式化表示为：

```math
Y_m^1 = [\text{MASK}]^{\otimes L}
```

```math
\tilde{Y}^i = \text{Classifier}(
\text{MDiffDecoder}(F_v, \text{Embedding}(Y_m^i))
)
```

```math
Y_m^{i+1} = \mathcal{M}_I(\tilde{Y}^i)
```

最终输出：

```math
\tilde{Y}^K
```

论文中最终主要采用：

```text
K = 3
```

即三步去噪完成识别。

---

## 11. 方法直观理解

MDiff4STR 可以理解为一种“反复校对式”的文本识别方法。

传统自回归模型像是在逐字写：

```text
已经写了 A R，下一个字符是什么？
```

普通并行模型像是一次性猜：

```text
这张图里的文本可能是 ARTE。
```

MDiff4STR 更像是反复检查和修改：

```text
先猜一遍整串文本；
把不确定的地方擦掉重写；
即使某些字符看起来确定，也要学会判断它是否真的正确；
最后得到更可靠的识别结果。
```

因此，MDiff4STR 具有以下优势：

1. 不依赖固定的从左到右生成顺序；
2. 可以利用双向上下文；
3. 可以多步修正错误；
4. 可以纠正高置信但错误的 token；
5. 在准确率和推理速度之间取得较好平衡。

---

## 12. 方法核心总结

MDiff4STR 的方法可以概括为：

> 以图像特征为条件，使用 Mask Diffusion Decoder 对被 mask 或被替换污染的文本 token 进行多步去噪与纠错；通过训练阶段的多种 mask 策略缓解 training-inference noising gap，并通过 Token-Replacement Noise 提升模型纠正高置信错误 token 的能力。

其核心设计包括：

| 模块                          | 作用                         |
| --------------------------- | -------------------------- |
| SVTRv2 Visual Encoder       | 提取文本图像的视觉特征                |
| Mask Diffusion Decoder      | 在图像条件下恢复文本 token           |
| Training Mask Strategies    | 缓解训练与推理噪声不一致               |
| Low-Confidence Remask       | 重新预测低置信度 token             |
| Block Low-Confidence Remask | 避免 confidence trap，提高推理稳定性 |
| Token-Replacement Noise     | 训练模型纠正错误 token             |
| Denoising Loss              | 学习补全被 mask 的字符             |
| Correction Loss             | 学习修正 corrupted sequence    |

最终，MDiff4STR 将 STR 解码从传统的“顺序生成”扩展为“多步全向去噪与纠错”，这是其方法设计的核心价值。
