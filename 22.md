1. 工作基本信息

SVTRv2 的论文题目是 “SVTRv2: CTC Beats Encoder-Decoder Models in Scene Text Recognition”，发表于 ICCV 2025。作者包括 Yongkun Du、Zhineng Chen、Hongtao Xie、Caiyan Jia、Yu-Gang Jiang，单位包括复旦大学可信具身智能研究院、中国科学技术大学、北京交通大学等。论文代码开源在 OpenOCR 项目中。

这篇工作的核心定位是：

让 CTC-based 场景文本识别模型在复杂场景下也能超过 Encoder-Decoder-based Text Recognition Models，同时保持 CTC 模型推理快、结构简单、部署友好的优势。

它不是完整 OCR 系统，而是文本识别模型，通常接在检测模型之后，对已经裁剪出的 text crop / text line 做识别。

2. 背景与动机

场景文本识别 STR，大体可以分为两类：

1. CTC-based 方法
   Visual Encoder + CTC aligned classifier

2. Encoder-Decoder-based 方法，简称 EDTR
   Visual Encoder + Attention / Transformer Decoder

CTC 方法的优势是结构简单、推理快、适合工业 OCR 部署；缺点是面对弯曲、旋转、多方向、艺术字、遮挡、中文等复杂场景时，通常不如 encoder-decoder 模型。论文认为 CTC 模型弱主要有两个原因：第一，CTC 默认文本特征序列接近从左到右排列，但不规则文本经常破坏这种顺序；第二，CTC 直接对视觉特征分类，本身缺少语言上下文建模能力。

而 EDTR 虽然精度强，但也有明显工业缺点：自回归 decoder 通常逐字符解码，推理慢；很多 attention-based 识别器默认最大识别长度为 25，长文本识别受限；一些 rectification 模块还会依赖固定输出尺寸，例如 32×128，并不适合所有文本形态。

所以 SVTRv2 的核心问题是：

能不能保留 CTC 的快速推理优势，同时补齐 CTC 对不规则文本和语言上下文建模不足的问题？

3. 核心贡献

SVTRv2 的贡献可以概括为四点。

第一，提出 MSR，Multi-Size Resizing。传统识别模型把所有文本 crop resize 到固定尺寸，例如 32×128，但这会对短文本、竖高文本、弯曲文本、极长文本造成不必要的形变。SVTRv2 根据文本长宽比选择不同输入尺寸，从源头减少图像几何失真。

第二，提出 FRM，Feature Rearrangement Module。它把二维视觉特征重新排列成更符合文本阅读顺序的一维序列，从而缓解 CTC 对齐困难。尤其是旋转、多方向、弯曲文本，原始特征排列和真实字符顺序不一致，FRM 就是专门解决这个问题。

第三，提出 SGM，Semantic Guidance Module。SGM 在训练阶段利用目标字符左右两侧的字符串上下文，引导视觉模型把语言上下文信息融入视觉特征中；推理时 SGM 被删除，所以不增加推理成本。

第四，构建了更公平的训练评测协议。论文指出一些大规模真实训练集和 Union14M-Benchmark 存在重叠，可能造成数据泄漏，因此提出 U14M-Filter，并基于该数据重新训练和评测 24 个主流 STR 方法。

4. 方法细节
4.1 整体架构

SVTRv2 的整体流程可以理解为：

输入 text crop
   ↓
MSR：根据长宽比选择合适 resize 尺寸
   ↓
Visual Encoder：三阶段特征提取，包含 local/global mixing
   ↓
FRM：将 2D visual feature 重排为符合阅读顺序的 1D sequence
   ↓
CTC classifier
   ↓
CTC decode 输出文本

训练时还会额外接入 SGM：

Visual Feature
   ├── FRM → CTC Loss
   └── SGM → Semantic Guidance Loss

推理时：

Visual Encoder + FRM + CTC

SGM 会被去掉，因此 SVTRv2 推理阶段仍然是纯 CTC 模型。

4.2 MSR：Multi-Size Resizing

传统 STR 模型常把所有文本图像 resize 成固定尺寸，例如：

32 × 128

问题是，不同文本 crop 的长宽比差异很大。比如短词、竖排文本、弯曲文本、长字符串，如果强行压缩到同一个尺寸，会出现字符变形、压扁、拉伸，导致后续视觉特征变差。

SVTRv2 按长宽比 R = W / H 分桶 resize：

长宽比范围	resize 尺寸
R < 1.5	64 × 64
1.5 ≤ R < 2.5	48 × 96
2.5 ≤ R < 3.5	40 × 112
R ≥ 3.5	32 × floor(R) × 32

也就是说，前三类用固定尺寸方便 batch 训练，第四类针对长文本使用动态宽度，减少长文本被严重压缩的问题。论文明确指出，这样可以尽量保持文本原始长宽比，降低 resize 造成的失真。

4.3 Visual Encoder：局部 + 全局混合

SVTRv2 继承 SVTR 的“单视觉模型”思想，但做了结构调整。它包含三阶段 visual feature extraction，每个阶段由若干 mixing blocks 组成。SVTRv2 使用两类 mixing：

Local Mixing：建模局部字符结构
Global Mixing：建模全局字符关系

Local Mixing 不再使用原 SVTR 中的 window attention，而是改为连续两个 grouped convolution。这样更适合捕捉字符边缘、笔画、纹理等局部结构，同时也更容易适配多尺寸输入，因为 SVTRv2 不使用 absolute positional encoding。Global Mixing 则使用 MHSA 进行全局上下文建模。

这点很关键。文本识别既需要局部细节，也需要全局上下文：

局部：笔画、边缘、字符形状、局部纹理
全局：字符顺序、词级上下文、整体排布

SVTRv2 的 encoder 设计就是为了同时覆盖这两类信息。

4.4 FRM：Feature Rearrangement Module

FRM 是 SVTRv2 最重要的模块之一。

CTC 的基本假设是：输入特征序列和输出字符序列之间存在单调对齐关系。也就是说，模型看到的第 1 段特征大概率对应前面的字符，第 2 段特征对应后面的字符。

但不规则文本会破坏这个假设。例如：

弯曲文本
旋转文本
多方向文本
艺术字
倾斜招牌
竖排或局部错位文本

这些情况下，二维特征图上从左到右扫描得到的 sequence，不一定等于真实阅读顺序。FRM 的作用就是学习一个 soft mapping，把二维视觉特征重新排列为符合 CTC 对齐要求的一维序列。

FRM 分两步：

1. Horizontal Rearranging
   对每一行特征做横向重排，学习水平方向上的字符排列关系。

2. Vertical Rearranging
   引入 selecting token，通过 cross-attention 选择不同高度位置的有效特征，
   再形成最终的一维字符特征序列。

可以理解为：先在每一行内部找字符顺序，再在不同高度之间选择真正属于阅读路径的特征。这样比简单 flatten 特征图更适合弯曲、旋转、多方向文本。

4.5 SGM：Semantic Guidance Module

SGM 解决的是 CTC 模型缺少语言上下文的问题。

CTC 推理时是对每个 timestep 的视觉特征独立分类，然后再做 collapse。它不像 decoder 那样天然使用前后字符上下文。因此，某些模糊、遮挡、易混字符场景中，CTC 可能只看局部视觉，很难利用词语上下文纠错。

SVTRv2 的 SGM 做法是：训练时，对于目标字符 ci，取它左侧字符串和右侧字符串作为上下文：

left context:  c_{i-ls}, ..., c_{i-1}
right context: c_{i+1}, ..., c_{i+ls}

然后将这些字符串 embedding 成上下文表示，再用 attention 去视觉特征图中寻找与目标字符相关的位置，并预测目标字符。这样，视觉 encoder 被迫学习一种“视觉特征中包含上下文信息”的表示。

训练目标是：

L = λ1 * L_ctc + λ2 * L_sgm

其中 L_ctc 是 CTC loss，L_sgm 是左右上下文预测目标字符的交叉熵 loss，论文中 λ1 = 0.1，λ2 = 1。推理时 SGM 删除，只保留 CTC 路径。

SGM 的意义在于：

训练时借助语言上下文增强视觉特征，推理时不带语言 decoder，不增加推理耗时。

这和很多 EDTR 直接在推理阶段使用 decoder 或语言模型不同，SVTRv2 更适合工业部署。

5. 模型规模

SVTRv2 提供 Tiny、Small、Base 三档：

模型	Channels	Blocks	Heads
SVTRv2-T	[64, 128, 256]	[3, 6, 3]	[2, 4, 8]
SVTRv2-S	[96, 192, 384]	[3, 6, 3]	[3, 6, 12]
SVTRv2-B	[128, 256, 384]	[6, 6, 6]	[4, 8, 12]

SVTRv2-T / S / B 对应不同精度和速度需求，适合从移动端到服务端的不同部署场景。

6. 实验结果概要

论文评测覆盖很多场景，包括常见英文识别 benchmark、Union14M-Benchmark、遮挡文本 OST、长文本 LTB，以及中文文本识别 BCTR。常见 benchmark 包括 IC13、SVT、IIIT5K、IC15、SVTP、CUTE80；Union14M-Benchmark 包含 Curve、Multi-Oriented、Artistic、Contextless、Salient、Multi-Words、General 等复杂子集；LTB 包含长度 25 到 35 的长文本样本。

6.1 英文与复杂场景结果

在统一使用 U14M-Filter 训练后，SVTRv2-B 的主要结果是：

模型	Com Avg	U14M Avg	LTB	OST	参数量	FPS
SVTRv2-T	94.78	79.05	47.8	71.4	5.1M	201
SVTRv2-S	96.13	83.70	47.6	78.0	11.3M	189
SVTRv2-B	96.57	86.14	50.2	80.0	19.8M	143

对比代表性 EDTR 方法，MAERec 的 U14M Avg 是 85.17，参数量 35.7M，FPS 17.1；SVTRv2-B 的 U14M Avg 是 86.14，参数量 19.8M，FPS 143。论文总结为：相比 MAERec，SVTRv2-B 在 U14M 上高 0.97%，推理速度约快 8 倍。

相比 CPPD，SVTRv2-B 在 U14M 上提升 4.23%，同时速度还快 10% 以上。论文还指出，SVTRv2-B 在 15 个评测场景中有 9 个取得最好结果，整体上超过多数 EDTR。

6.2 不规则文本和长文本

SVTRv2 对 Curve 和 Multi-Oriented 这类不规则文本提升非常明显。论文指出，相比 SVTR-B，SVTRv2-B 在 Curve 和 MO 子集上分别提升 14.4% 和 44.5%。

这说明 SVTRv2 真正解决的是 CTC 模型原来最弱的地方：不规则文本下的阅读顺序与 CTC 对齐问题。

长文本方面，很多 attention decoder 默认最大长度 25，而 LTB 测试集长度为 25 到 35。论文显示，带 rectification 或 attention decoder 的替代方案在 LTB 上经常识别失败，而 SVTRv2 可以保留 CTC 对长序列更友好的优势。

6.3 中文识别结果

在中文 BCTR 数据集上，SVTRv2-B 的结果为：

模型	Scene	Web	Doc	HW	Avg	SceneL>25	Size
SVTRv2-T	77.8	78.8	99.2	62.0	79.45	47.8	6.8M
SVTRv2-S	81.1	81.2	99.3	65.0	81.64	50.0	14.0M
SVTRv2-B	83.5	83.3	99.5	67.0	83.31	52.8	22.5M

论文认为 SVTRv2-B 在中文识别上达到 SOTA，并且对中文长文本也有较好适应性。

6.4 预训练对比

在加入预训练并与一些大规模视觉语言预训练 EDTR 方法比较时，SVTRv2-B 的 Com Avg 达到 97.83，OST 达到 86.9，参数量 19.8M，FPS 143。论文指出，相比 CLIP4STR，SVTRv2 只使用约 14% 的参数量，并且速度约快 10 倍。

这说明 SVTRv2 不只是无预训练场景强，在结合预训练后也有较强扩展性。

7. 消融实验分析

SVTRv2 的消融实验比较清晰。

7.1 MSR 的作用

MSR 在不规则文本长宽比变化较大的场景中收益明显。论文提到，在 Curve 和 MO 的部分长宽比区间，MSR 相比固定 32×128 resize 分别带来 15.3% 和 5.2% 的提升，说明固定 resize 对复杂文本的形变确实会显著影响识别。

7.2 FRM 的作用

FRM 对 Multi-Oriented 文本很重要。论文消融显示，单独 horizontal rearranging 在 MO 上提升 2.03%，单独 vertical rearranging 提升 0.71%，二者结合提升 2.46%。这说明二维到一维的重排确实改善了 CTC 的序列对齐。

7.3 SGM 的作用

SGM 对遮挡文本尤其有效。论文中 SGM 在 Com 上提升 0.41%，在 U14M 上提升 2.28%，在 OST 遮挡文本上提升 5.11%。这说明当字符不完整、局部视觉信息不足时，语言上下文引导能够明显改善视觉特征表示。

7.4 Progressive ablation 的整体结论

论文的 progressive ablation 总结是：提高特征分辨率、用 Conv2 替代局部 attention、加入 MSR / FRM、加入 SGM，分别改善了不规则文本、艺术字、遮挡文本和语言上下文建模问题。最终 SVTRv2 在保留 CTC 推理效率的同时，显著提升了复杂文本识别能力。

8. 深度分析
8.1 SVTRv2 的本质：不是抛弃 CTC，而是补齐 CTC

很多 STR 论文提升复杂场景精度的方式是加 attention decoder、language model、rectification module，最终模型越来越重。SVTRv2 的思路不同，它不想把 CTC 改成 EDTR，而是分析 CTC 为什么弱，然后针对性补齐：

CTC 弱点 1：输入图像被 resize 后形变严重
解决：MSR

CTC 弱点 2：不规则文本破坏单调对齐
解决：FRM

CTC 弱点 3：缺少语言上下文
解决：SGM training-only guidance

这种方法很适合工业 OCR，因为它把复杂能力尽量放到训练阶段，而不是推理阶段。

8.2 FRM 是 SVTRv2 最关键的结构创新

MSR 是输入策略，SGM 是训练辅助，真正改变 CTC 特征序列生成方式的是 FRM。

传统 CTC 识别一般是：

2D feature map
   ↓
height pooling / flatten
   ↓
从左到右序列
   ↓
CTC

这对水平规则文本有效，但对弯曲、多方向文本不够。FRM 相当于学习一个软阅读路径：

2D feature map
   ↓
horizontal rearranging
   ↓
vertical selecting
   ↓
reading-order feature sequence
   ↓
CTC

因此，FRM 的意义不是简单增强特征，而是让特征排列方式更符合 CTC 对齐假设。

8.3 SGM 的设计很适合小模型训练

SGM 最大的价值是“训练时用语言，推理时不用语言”。这对小模型非常重要。

如果推理时接 decoder 或语言模型，会带来：

自回归延迟
部署复杂度
更高显存 / 内存占用
batch 推理效率下降
潜在语言幻觉

而 SGM 的做法是训练时通过左右字符串上下文引导视觉特征学习，推理时仍然走 CTC 并行解码。这和你前面梳理的 PP-OCRv6 中“训练时使用 NRTR 辅助，推理时只保留 CTC”的思想有相似之处。PP-OCRv6 的识别模型也是训练阶段用 CTC+NRTR，推理阶段只用 CTC，以兼顾上下文学习和推理效率。

8.4 SVTRv2 和 PP-OCRv6 的关系

二者都属于非 VLM 的 OCR 小模型路线，但侧重点不同。

维度	SVTRv2	PP-OCRv6
任务	文本识别 STR	完整 OCR 系统，检测 + 识别
核心目标	让 CTC 识别超过 EDTR	构建轻量工业 OCR 系统
关键模块	MSR、FRM、SGM	LCNetV4、RepLKFPN、LightSVTR、CTC+NRTR
推理方式	Visual Encoder + FRM + CTC	检测 DB + 识别 CTC
语言建模	SGM 训练引导，推理删除	NRTR 训练辅助，推理删除
工业价值	识别模型强，适合替换 OCR recognizer	端到端 OCR pipeline，更完整

可以说：

SVTRv2 重点解决“识别模型如何更强”；PP-OCRv6 重点解决“检测 + 识别整体系统如何更强、更快、更容易部署”。

9. 局限性

SVTRv2 虽然很强，但也有一些需要注意的点。

第一，它主要是识别模型，不解决文本检测问题。真实 OCR pipeline 中，如果检测框漏检、框偏、裁剪不完整，识别模型仍然会受影响。

第二，SGM 使用 ground-truth 左右字符串上下文做训练引导，因此对标注质量比较敏感。如果训练数据中存在错标、漏字符、特殊符号标注不一致，SGM 可能会学习到错误的上下文关系。

第三，虽然 SVTRv2 对不规则文本提升明显，但论文的 bad case 分析显示，剩余错误主要集中在模糊文本、艺术字、不完整文本，以及标注错误。论文统计在 Com 的错误样本中，模糊占 30.81%，艺术字占 24.24%，不完整文本占 31.82%。

第四，SVTRv2 的强性能依赖大规模真实数据训练和较严格的评测过滤。论文自己也强调真实数据对 STR 性能非常关键，并指出不同训练集与评测集重叠会影响公平性。

10. 对你在荣耀 OCR 实习的启发

结合你现在做 OCR 数据校验，SVTRv2 有几个可以借鉴的点。

第一，识别模型训练要重视 crop 形变问题。很多训练样本被统一 resize 到固定大小，这可能会让短文本、长文本、竖高文本、弯曲文本严重变形。你可以在数据校验中加入一个指标：

原始文本框长宽比
resize 后长宽比
字符是否被压扁 / 拉伸
长文本是否被过度压缩

这可以帮助判断训练样本是否适合当前 recognizer 的输入策略。

第二，不规则文本需要单独分层评估。SVTRv2 的核心提升来自 Curve、Multi-Oriented、Occluded、Artistic、Long text 等场景。你在荣耀内部也可以把 OCR 回流数据按场景分桶：

水平规则文本
旋转文本
弯曲文本
屏幕文本
艺术字 / Logo 字
模糊文本
遮挡 / 裁剪不完整文本
长字符串 / SN 码 / 型号

然后分别统计 baseline、小模型、大模型 teacher 的错误类型。

第三，大模型可以用于发现 SGM 类训练信号的错误样本。SGM 对标注质量敏感，而你现在用 GLM-OCR、dots.mOCR 做校验，刚好可以筛出：

字符错标
漏字符
多字符
相似字符混淆
特殊符号不一致
大小写 / 空格 / 标点规则不一致

这些样本如果不处理，会影响 CTC 和 SGM 这类语言上下文引导训练。

第四，可以把 SVTRv2 思想写成实习项目亮点：

针对 OCR 识别模型在复杂文本场景下的性能瓶颈，参考 SVTRv2 对 CTC 模型的分析，将回流数据按文本形态和识别难度进行分层，包括长宽比异常、旋转弯曲、遮挡模糊、长文本和特殊字符等类型；结合多 Teacher OCR 结果与原始标注进行一致性校验，识别 resize 形变严重、标注不可靠和上下文冲突样本，从而提升轻量识别模型训练数据质量。

11. 汇报版总结

SVTRv2 是 ICCV 2025 的场景文本识别工作，目标是在保持 CTC 模型高速、简单、易部署优势的同时，补齐 CTC 在不规则文本和语言上下文建模上的不足。它提出 MSR 根据文本长宽比自适应 resize，减少固定尺寸输入带来的几何形变；提出 FRM 将二维视觉特征重排为符合阅读顺序的一维序列，缓解旋转、弯曲、多方向文本下的 CTC 对齐困难；提出 SGM 在训练阶段利用左右字符串上下文引导视觉模型学习语言信息，推理时删除该模块，因此不增加推理成本。实验上，SVTRv2-B 在 U14M、OST、LTB、中文识别等多类场景中超过多数 EDTR，同时参数量和速度更适合工业部署。整体来看，SVTRv2 的价值在于证明 CTC 路线并没有过时，只要解决输入形变、特征对齐和语言上下文三个关键问题，CTC-based recognizer 仍然可以在复杂场景下达到甚至超过 encoder-decoder 模型的效果。
