1. 团队与论文基本信息

论文题目：MDiff4STR: Mask Diffusion Model for Scene Text Recognition。
会议：AAAI 2026，官方页面列在 AAAI Technical Track on Computer Vision II，论文页码为 3705–3713，DOI 为 10.1609/aaai.v40i5.37370；arXiv 页面标注为 AAAI 2026 Oral。

作者与单位：Yongkun Du、Miaomiao Zhao、Songlin Fan、Zhineng Chen、Caiyan Jia、Yu-Gang Jiang。主要来自复旦大学 Institute of Trustworthy Embodied AI、北京交通大学计算机学院，以及 China Mobile Shanghai ICT Co., Ltd.；论文 PDF 标注 Yongkun Du 和 Miaomiao Zhao 为共同一作，Zhineng Chen 为通讯作者。

研究对象：场景文本识别，Scene Text Recognition，简称 STR。论文目标是把 Mask Diffusion Model，MDM 引入 STR，用扩散式、多步去噪的方式识别图像中的文本。

2. 背景与研究动机

STR 是 OCR 系统中的基础任务，但真实场景里的文本经常存在弯曲、变形、方向变化、遮挡、模糊、艺术字体等问题，因此模型不仅要看清图像，还需要较强的语言上下文推理能力。

已有 STR 方法大致可以分为几类：自回归模型 ARM 准确率强，但逐字符解码导致速度慢；并行解码模型 PDM 推理快，但缺少字符间上下文建模；BERT-like refinement 模型 ReM 先并行预测再 refine，但容易受初始预测错误影响。论文认为，MDM 通过从部分 mask 的序列中逐步恢复文本，可以建模更灵活的全向依赖，有潜力同时兼顾准确率与效率。

不过，直接把 vanilla MDM 用到 STR 上效果并不理想：它虽然推理更高效，但准确率落后于 ARM。作者进一步归因于两个关键问题：第一，训练时通常是随机 mask，而推理时是全 mask 起步并使用结构化 remask，存在 training-inference noising gap；第二，模型会对错误 token 产生过高置信度，导致低置信度 remask 机制难以发现并修正错误。

3. 核心贡献

第一，首次系统性地将 MDM 引入 STR。
论文把文本识别从“从左到右逐字符生成”转为“从 mask 序列中逐步去噪恢复文本”，使 STR 可以使用更灵活的全向语言建模。

第二，提出面向 STR 的训练噪声策略。
为缓解训练与推理噪声分布不一致，作者在随机 mask 之外，引入与推理阶段一致的多种 mask/remask 策略，包括 full mask、左右 remask、cloze remask、low-confidence remask、block remask 等，使模型训练时就见过推理时会遇到的噪声形态。

第三，提出 Token-Replacement Noise，TRN。
TRN 不只是把字符 mask 掉，而是随机替换成错误字符，让模型学习识别和纠正“看起来很确定但其实错了”的 token，从而缓解 MDM 推理中的过度自信问题。

第四，实验上取得更好的精度—速度权衡。
论文报告 MDiff4STR 在多个 STR benchmark 上超过已有强模型，并且专门的 low-confidence denoising 推理只需要 3 步去噪即可超过 ARM，同时推理速度约为 ARMbase 的 3 倍。

4. 方法细节

整体架构包括两个部分：图像侧使用 SVTRv2 作为视觉编码器提取图像特征；文本侧把字符序列加噪后输入 Mask Diffusion Decoder，decoder 在视觉特征条件下恢复原始文本。

推理过程是一个 K-step denoising diffusion process。第一步将整个文本序列设为 [MASK]，表示完全未知；随后进行 K−1 次 remask 与再预测，逐步得到最终识别结果。根据 remask 策略不同，MDiff4STR 可以模拟或实现多种解码范式：parallel decoding，auto-regressive decoding，BERT-like refinement，以及论文重点提出的 low-confidence remask 和 block low-confidence remask。

训练时，vanilla MDM 只使用随机 mask，会导致训练阶段和推理阶段噪声模式不一致。MDiff4STR 因此在训练时从七类策略中均匀采样：随机 mask 加上六种与推理相关的 mask/remask 策略，让训练分布更贴近推理分布。

TRN 的做法是随机把原始字符替换成其他字符，构造 corrupted sequence，并要求模型恢复正确文本。这样模型不只会补全 [MASK]，也会学会“质疑并修正已有 token”，尤其是那些在推理中可能置信度很高但实际错误的 token。

训练目标由两部分组成：denoising loss 主要监督被 mask 的位置；correction loss 对整个序列进行监督，因为推理时模型并不知道哪些 token 是错的。最终损失是二者相加。

5. 实验

实验覆盖英文和中文文本识别。英文部分训练在 U14M-Filter，并在常见 benchmark Com、Union14M-Benchmark，以及遮挡文本数据集 OST 上评测；中文部分使用 BCTR，包含 Scene、Web、Document、Hand-Writing 四类场景。评价指标是 word accuracy。

消融实验显示，去噪步数 K 和 decoder 层数 N 影响精度和速度。论文最终选择 K=3、N=6 作为折中配置；在该设置下，MDiff4STR-BLC 在 Com、U14M、OST 上分别达到 97.30、88.44、84.25，推理时间为 19.21ms。

训练 mask 策略有效缓解 noising gap。加入六种训练 mask/remask 策略后，五种 denoising 范式在 Com、U14M、OST 上都有提升，平均提升分别为 0.43%、1.68%、2.17%；其中更困难的 U14M 和 OST 受益更明显。

TRN 对性能提升更明显。论文报告，TRN 相比 vanilla MDM 在 Com、U14M、OST 上分别带来 0.91%、2.93%、4.58% 的平均准确率提升，并使 MDiff4STR-LC/BLC 在仅 3 步去噪下超过 ARMbase。

与 SOTA 比较中，从零训练的 MDiff4STR-B-BLC 在 Com、U14M、OST 上分别达到 97.30、88.44、84.3；论文称其相较此前最好结果分别提升 0.73%、2.30%、4.30%，其中 OST 的提升说明全向上下文建模对遮挡文本尤其有效。

在大规模预训练设置下，MDiff4STR-BLC 达到 Com 98.02、OST 87.4；中文 BCTR 上，MDiff4STR-B-BLC 的平均准确率达到 84.25，也超过此前方法，说明方法对非拉丁文字也有一定泛化能力。

6. 小结

MDiff4STR 的核心价值在于：它不是简单把扩散模型套到 OCR 上，而是提出了一种新的 STR 解码范式——从全 mask 或部分噪声文本出发，通过多步去噪和纠错恢复文本。相比 ARM，它不局限于左到右建模；相比普通并行模型，它保留了上下文推理能力；相比 BERT-like refinement，它通过 TRN 增强了对错误 token 的修正能力。

可以把这篇论文概括为：用 Mask Diffusion 做场景文本识别，并通过“训练推理噪声对齐 + token 替换纠错”解决 MDM 在 STR 中的两个核心短板，最终在准确率和速度之间取得更优平衡。
